 Serilog is a diagnostic logging library for .NET applications. It is easy to set up, has a clean API, and runs on all recent .NET platforms. While it\textquotesingle{}s useful even in the simplest applications, Serilog\textquotesingle{}s support for structured logging shines when instrumenting complex, distributed, and asynchronous applications and systems.

\href{https://serilog.net}{\texttt{ }}

Like many other libraries for .NET, Serilog provides diagnostic logging to \href{https://github.com/serilog/serilog-sinks-file}{\texttt{ files}}, the \href{https://github.com/serilog/serilog-sinks-console}{\texttt{ console}}, and \href{https://github.com/serilog/serilog/wiki/Provided-Sinks}{\texttt{ many other outputs}}.

\DoxyCodeLine{\textcolor{keyword}{using\ }var\ log\ =\ \textcolor{keyword}{new}\ LoggerConfiguration()}
\DoxyCodeLine{\ \ \ \ .WriteTo.Console()}
\DoxyCodeLine{\ \ \ \ .WriteTo.File(\textcolor{stringliteral}{"{}log.txt"{}})}
\DoxyCodeLine{\ \ \ \ .CreateLogger();}
\DoxyCodeLine{}
\DoxyCodeLine{log.Information(\textcolor{stringliteral}{"{}Hello,\ Serilog!"{}});}

\end{DoxyCode}


Unlike other logging libraries, Serilog is built from the ground up to record {\itshape structured event data}.


\begin{DoxyCode}{0}
\DoxyCodeLine{var\ position\ =\ \textcolor{keyword}{new}\ \{\ Latitude\ =\ 25,\ Longitude\ =\ 134\ \};}
\DoxyCodeLine{var\ elapsedMs\ =\ 34;}
\DoxyCodeLine{}
\DoxyCodeLine{log.Information(\textcolor{stringliteral}{"{}Processed\ \{@Position\}\ in\ \{Elapsed\}\ ms"{}},\ position,\ elapsedMs);}

\end{DoxyCode}


Serilog uses \href{https://messagetemplates.org}{\texttt{ message templates}}, a simple DSL that extends .NET format strings with {\itshape named} as well as positional parameters. Instead of formatting events immediately into text, Serilog captures the values associated with each named parameter.

The example above records two properties, {\ttfamily Position} and {\ttfamily Elapsed}, in the log event. The {\ttfamily @} operator in front of {\ttfamily Position} tells Serilog to {\itshape serialize} the object passed in, rather than convert it using {\ttfamily To\+String()}. Serilog\textquotesingle{}s deep and rich support for structured event data opens up a huge range of diagnostic possibilities not available when using traditional loggers.

Rendered into \href{https://github.com/serilog/serilog-formatting-compact}{\texttt{ JSON format}} for example, these properties appear alongside the timestamp, level, and message like\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\{"{}Position"{}:\ \{"{}Latitude"{}:\ 25,\ "{}Longitude"{}:\ 134\},\ "{}Elapsed"{}:\ 34\}}

\end{DoxyCode}


Back-\/ends that are capable of recording structured event data make log searches and analysis possible without log parsing or regular expressions.

Supporting structured data doesn\textquotesingle{}t mean giving up text\+: when Serilog writes events to files or the console, the template and properties are rendered into friendly human-\/readable text just like a traditional logging library would produce\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{09:14:22\ [INF]\ Processed\ \{"{}Latitude"{}:\ 25,\ "{}Longitude"{}:\ 134\}\ in\ 34\ ms.}

\end{DoxyCode}

\hypertarget{md_packages_2_serilog_83_81_81_2_r_e_a_d_m_e_autotoc_md152}{}\doxysubsection{\texorpdfstring{Features}{Features}}\label{md_packages_2_serilog_83_81_81_2_r_e_a_d_m_e_autotoc_md152}

\begin{DoxyItemize}
\item {\bfseries{Community-\/backed and actively developed}}
\item Format-\/based logging API with familiar \href{https://github.com/serilog/serilog/wiki/Configuration-Basics\#minimum-level}{\texttt{ levels}} like {\ttfamily Debug}, {\ttfamily Information}, {\ttfamily Warning}, {\ttfamily Error}, and so-\/on
\item Discoverable C\# configuration syntax and optional \href{https://github.com/serilog/serilog-settings-appsettings}{\texttt{ XML}} or \href{https://github.com/serilog/serilog-settings-configuration}{\texttt{ JSON}} configuration support
\item Efficient when enabled, extremely low overhead when a logging level is switched off
\item Best-\/in-\/class .NET Core support, including \href{https://github.com/serilog/serilog-aspnetcore}{\texttt{ rich integration with ASP.\+NET Core}}
\item Support for a \href{https://github.com/serilog/serilog/wiki/Provided-Sinks}{\texttt{ comprehensive range of sinks}}, including files, the console, on-\/premises and cloud-\/based log servers, databases, and message queues
\item Sophisticated \href{https://github.com/serilog/serilog/wiki/Enrichment}{\texttt{ enrichment}} of log events with contextual information, including scoped ({\ttfamily Log\+Context}) properties, thread and process identifiers, and domain-\/specific correlation ids such as {\ttfamily Http\+Request\+Id}
\item Zero-\/shared-\/state {\ttfamily Logger} objects, with an optional global static {\ttfamily Log} class
\item Format-\/agnostic logging pipeline that can emit events in plain text, JSON, in-\/memory {\ttfamily Log\+Event} objects (including \href{https://github.com/serilog/serilog-sinks-observable}{\texttt{ Rx pipelines}}) and other formats
\end{DoxyItemize}\hypertarget{md_packages_2_serilog_83_81_81_2_r_e_a_d_m_e_autotoc_md153}{}\doxysubsection{\texorpdfstring{Getting started}{Getting started}}\label{md_packages_2_serilog_83_81_81_2_r_e_a_d_m_e_autotoc_md153}
Serilog is installed \href{https://nuget.org/packages/serilog}{\texttt{ from Nu\+Get}}. To view log events, one or more sinks need to be installed as well, here we\textquotesingle{}ll use the pretty-\/printing console sink, and a rolling file set\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{dotnet\ add\ package\ Serilog}
\DoxyCodeLine{dotnet\ add\ package\ Serilog.Sinks.Console}
\DoxyCodeLine{dotnet\ add\ package\ Serilog.Sinks.File}

\end{DoxyCode}
