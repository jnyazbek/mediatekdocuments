 A Serilog sink that writes log events to the Windows Console or an ANSI terminal via standard output. Coloring and custom themes are supported, including ANSI 256-\/color themes on mac\+OS, Linux and Windows 10. The default output is plain text; JSON formatting can be plugged in using a package such as \href{https://github.com/serilog/serilog-formatting-compact}{\texttt{ {\itshape Serilog.\+Formatting.\+Compact}}}.\hypertarget{md_packages_2_serilog_8_sinks_8_console_85_80_81_2_r_e_a_d_m_e_autotoc_md171}{}\doxysubsection{\texorpdfstring{Getting started}{Getting started}}\label{md_packages_2_serilog_8_sinks_8_console_85_80_81_2_r_e_a_d_m_e_autotoc_md171}
To use the console sink, first install the \href{https://nuget.org/packages/serilog.sinks.console}{\texttt{ Nu\+Get package}}\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{dotnet\ add\ package\ Serilog.Sinks.Console}

\end{DoxyCode}


Then enable the sink using {\ttfamily Write\+To.\+Console()}\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{Log.Logger\ =\ \textcolor{keyword}{new}\ LoggerConfiguration()}
\DoxyCodeLine{\ \ \ \ .WriteTo.Console()}
\DoxyCodeLine{\ \ \ \ .CreateLogger();}
\DoxyCodeLine{\ \ \ \ }
\DoxyCodeLine{Log.Information(\textcolor{stringliteral}{"{}Hello,\ world!"{}});}

\end{DoxyCode}


Log events will be printed to {\ttfamily STDOUT}\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{[12:50:51\ INF]\ Hello,\ world!}

\end{DoxyCode}
\hypertarget{md_packages_2_serilog_8_sinks_8_console_85_80_81_2_r_e_a_d_m_e_autotoc_md172}{}\doxysubsection{\texorpdfstring{Themes}{Themes}}\label{md_packages_2_serilog_8_sinks_8_console_85_80_81_2_r_e_a_d_m_e_autotoc_md172}
The sink will colorize output by default\+:



Themes can be specified when configuring the sink\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{.WriteTo.Console(theme:\ AnsiConsoleTheme.Code)}

\end{DoxyCode}


The following built-\/in themes are available\+:


\begin{DoxyItemize}
\item {\ttfamily Console\+Theme.\+None} -\/ no styling
\item {\ttfamily System\+Console\+Theme.\+Literate} -\/ styled to replicate {\itshape Serilog.\+Sinks.\+Literate}, using the {\ttfamily System.\+Console} coloring modes supported on all Windows/.NET targets; {\bfseries{this is the default when no theme is specified}}
\item {\ttfamily System\+Console\+Theme.\+Grayscale} -\/ a theme using only shades of gray, white, and black
\item {\ttfamily Ansi\+Console\+Theme.\+Literate} -\/ an ANSI 256-\/color version of the "{}literate"{} theme
\item {\ttfamily Ansi\+Console\+Theme.\+Grayscale} -\/ an ANSI 256-\/color version of the "{}grayscale"{} theme
\item {\ttfamily Ansi\+Console\+Theme.\+Code} -\/ an ANSI 256-\/color Visual Studio Code-\/inspired theme
\item {\ttfamily Ansi\+Console\+Theme.\+Sixteen} -\/ an ANSI 16-\/color theme that works well with both light and dark backgrounds
\end{DoxyItemize}

Adding a new theme is straightforward; examples can be found in the \href{https://github.com/serilog/serilog-sinks-console/blob/dev/src/Serilog.Sinks.Console/Sinks/SystemConsole/Themes/SystemConsoleThemes.cs}{\texttt{ {\ttfamily System\+Console\+Themes}}} and \href{https://github.com/serilog/serilog-sinks-console/blob/dev/src/Serilog.Sinks.Console/Sinks/SystemConsole/Themes/AnsiConsoleThemes.cs}{\texttt{ {\ttfamily Ansi\+Console\+Themes}}} classes.\hypertarget{md_packages_2_serilog_8_sinks_8_console_85_80_81_2_r_e_a_d_m_e_autotoc_md173}{}\doxysubsection{\texorpdfstring{Output templates}{Output templates}}\label{md_packages_2_serilog_8_sinks_8_console_85_80_81_2_r_e_a_d_m_e_autotoc_md173}
The format of events to the console can be modified using the {\ttfamily output\+Template} configuration parameter\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{.WriteTo.Console(}
\DoxyCodeLine{\ \ \ \ outputTemplate:\ \textcolor{stringliteral}{"{}[\{Timestamp:HH:mm:ss\}\ \{Level:u3\}]\ \{Message:lj\}\{NewLine\}\{Exception\}"{}})}

\end{DoxyCode}


The default template, shown in the example above, uses built-\/in properties like {\ttfamily Timestamp} and {\ttfamily Level}. Properties from events, including those attached using \href{https://github.com/serilog/serilog/wiki/Enrichment}{\texttt{ enrichers}}, can also appear in the output template.\hypertarget{md_packages_2_serilog_8_sinks_8_console_85_80_81_2_r_e_a_d_m_e_autotoc_md174}{}\doxysubsection{\texorpdfstring{JSON output}{JSON output}}\label{md_packages_2_serilog_8_sinks_8_console_85_80_81_2_r_e_a_d_m_e_autotoc_md174}
The sink can write JSON output instead of plain text. {\ttfamily Compact\+Json\+Formatter} or {\ttfamily Rendered\+Compact\+Json\+Formatter} from \href{https://github.com/serilog/serilog-formatting-compact}{\texttt{ Serilog.\+Formatting.\+Compact}} is recommended\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{dotnet\ add\ package\ Serilog.Formatting.Compact}

\end{DoxyCode}


Pass a formatter to the {\ttfamily Console()} configuration method\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{.WriteTo.Console(\textcolor{keyword}{new}\ RenderedCompactJsonFormatter())}

\end{DoxyCode}


Output theming is not available when custom formatters are used.\hypertarget{md_packages_2_serilog_8_sinks_8_console_85_80_81_2_r_e_a_d_m_e_autotoc_md175}{}\doxysubsection{\texorpdfstring{XML {\ttfamily \texorpdfstring{$<$}{<}app\+Settings\texorpdfstring{$>$}{>}} configuration}{XML {\ttfamily \texorpdfstring{$<$}{<}app\+Settings\texorpdfstring{$>$}{>}} configuration}}\label{md_packages_2_serilog_8_sinks_8_console_85_80_81_2_r_e_a_d_m_e_autotoc_md175}

\begin{DoxyCode}{0}
\DoxyCodeLine{dotnet\ add\ package\ Serilog.Settings.AppSettings}

\end{DoxyCode}

Console logging is synchronous and this can cause bottlenecks in some deployment scenarios. For high-\/volume console logging, consider using \href{https://github.com/serilog/serilog-sinks-async}{\texttt{ {\itshape Serilog.\+Sinks.\+Async}}} to move console writes to a background thread\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{comment}{//\ dotnet\ add\ package\ serilog.sinks.async}}
\DoxyCodeLine{}
\DoxyCodeLine{Log.Logger\ =\ \textcolor{keyword}{new}\ LoggerConfiguration()}
\DoxyCodeLine{\ \ \ \ .WriteTo.Async(wt\ =>\ wt.Console())}
\DoxyCodeLine{\ \ \ \ .CreateLogger();}

\end{DoxyCode}
